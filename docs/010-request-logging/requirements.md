# Issue #10: Request Logging

## Problem Statement

現在、md-serverにはHTTPリクエストに関するログ出力機能がない。サーバーへのアクセス状況を把握するために、リクエストログを標準出力に出力する機能が必要。

## Requirements

- HTTPリクエストを受けた際に、INFOレベルのログを標準出力に出力する
- ログには以下の情報を含める:
  - タイムスタンプ (ISO 8601形式)
  - ログレベル (`INFO`)
  - HTTPメソッド (`GET`)
  - リクエストパス
  - レスポンスステータスコード
  - レスポンスタイム (ms)

## Constraints

- 外部ログライブラリは使用せず、自前で実装する
- ログはJSON形式ではなく、人間が読みやすいテキスト形式で出力する
- Expressのミドルウェアとして実装する
- 既存のテストに影響を与えない

## Acceptance Criteria

- すべてのHTTPリクエストに対してINFOレベルのログが標準出力に出力される
- ログにタイムスタンプ、メソッド、パス、ステータスコード、レスポンスタイムが含まれる
- 既存のテストがすべてパスする
- ログミドルウェア自体のテストが存在する

## User Stories

- サーバー管理者として、リクエストログを確認してアクセス状況を把握したい
- 開発者として、デバッグのためにリクエストの詳細を確認したい
